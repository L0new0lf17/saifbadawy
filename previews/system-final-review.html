Limits & Constraints
Honest assessment — nothing hidden
Make.com
⚙️
10,000 ops/month cap
You're using ~3,314 ops/month (33%). You'd need traffic to triple before approaching the limit. If you ever hit it, Make.com pauses scenarios until next month or you upgrade to Pro ($16/mo for 20k ops).
✓ Not a concern at current scale
⏱
Gmail watcher polls every 15 minutes
Emails are not instant — they're picked up within 15 minutes of arriving. For a portfolio system this is completely acceptable. Contact forms and Calendly use webhooks so those are instant.
⚠ Minor — 15 min delay on emails only
Notion
🔄
3 API requests per second
Notion's only real limit. Your system makes ~550 calls/month spread across all hours. You'd need multiple visitors simultaneously all using tools at the exact same second to approach this. Essentially impossible at your scale.
✓ Never going to be an issue
📅
7-day page history on free plan
Notion free plan only keeps 7 days of edit history. This means if you accidentally delete or change something, you can only restore it from within the last 7 days. Your actual data is never deleted — only the version history is limited.
⚠ Minor — backup manually monthly if concerned
LinkedIn Insight Tag
🔗
Requires LinkedIn Campaign Manager access
The Insight Tag needs a LinkedIn Company Page and Campaign Manager account to see company visitor data. For a personal portfolio, you can still install it and see aggregate data, but individual company identification requires running LinkedIn ads (even $1 minimum). Without that, you get traffic stats but not company names.
⚠ Company-level data requires minimal ad spend or Campaign Manager access
Applications Logger
📤
Online portal applications may not auto-log
If you apply via LinkedIn Easy Apply, a company's careers portal, or any system that doesn't send a confirmation email, Make.com won't catch it. You'd add those manually — 30 seconds per application. Any application that sends a confirmation email to Gmail is caught automatically.
⚠ Portal applications without email confirmations need manual entry
EmailJS
📧
200 emails/month on free plan
EmailJS free tier allows 200 contact form submissions per month. For a personal portfolio this is more than enough — that's 6-7 form submissions per day. If you ever exceed this, EmailJS Personal plan is $9/mo for 1,000 emails. Very unlikely to hit this.
✓ More than enough for a portfolio
Calendly
📅
1 event type on free plan
Calendly free only allows one active event type (e.g. "30-min Discovery Call"). If you want multiple booking types (e.g. consulting vs job interview vs quick chat), you'd need Calendly Standard at $10/mo. For now, one type is perfectly sufficient.
✓ One event type is enough to start
Summary: The only real constraints worth noting are the 15-minute Gmail polling delay (minor), Notion's 7-day history limit (minor), the LinkedIn Insight Tag needing Campaign Manager for company names, and online portal applications needing manual logging if there's no confirmation email. None of these affect the core system meaningfully. Everything else is effectively unlimited at your scale.

One-time setup. Estimated total time: 3–5 hours spread across a few sessions.
01
Create Notion Workspace & 10 Databases
~45 min First
Set up all 10 databases with the correct properties, views, and relations between them. Lead Pipeline links to Companies, People, Bookings, Applications. Bookings links to Interview Notes. This is the foundation everything else builds on.
👤 Lead Pipeline 👁 Visitor Log 📆 Bookings ✅ Follow-up Tasks 💬 Tool Interactions 📋 Weekly Digest 🏢 Companies 🧑 People 📤 Applications 🎤 Interview Notes
02
Create Notion Integration & API Key
~10 min
Create a Notion integration at notion.so/my-integrations. This generates an API key that lets the website JS and Make.com both write to your Notion databases. Share each database with the integration.
03
Set Up Telegram Bot
~10 min
Message @BotFather on Telegram → create a new bot → get the bot token. Start a conversation with your bot → get your Chat ID. These two values are what Make.com uses to send messages directly to your phone.
04
Add Direct Notion JS to Website
~30 min
Add JavaScript to index_main.html that fires on chatbot messages, ROI submissions, 90-day plan generations, and page visits. Each event POSTs directly to the Notion API — bypassing Make.com entirely, costing zero ops.
Visitor log on page load Chatbot message logger ROI Calculator logger 90-Day Plan logger
05
Create Make.com Account & Build 10 Scenarios
~2 hrs Biggest step
Sign up for Make.com Core ($9/mo). Build each scenario one by one, connecting Gmail, Notion, Telegram, Calendly, and your website webhooks. Each scenario is a visual drag-and-drop flow — no coding required.
📧 Email Router 📅 Calendly Handler 📝 Contact Form 🎯 High-Intent Visitor 🚨 Site Down Alert 📊 Weekly Digest 🧑 Auto-Create People 🔄 Lead Status Updates 📤 Application Logger 🎤 Interview Notes Page
06
Set Up UptimeRobot
~10 min
Create a free UptimeRobot account. Add saifbadawy.com as a monitor (HTTP check, every 5 minutes). Add the Make.com webhook URL as an alert contact so it fires automatically when your site goes down.
07
Verify Google Analytics 4 & Search Console
~15 min
Confirm GA4 tracking is firing correctly. Submit your sitemap to Search Console if not already done. These should already be partially set up — just verify data is flowing.
08
End-to-End Test
~30 min Final
Submit a test contact form → check Notion and Telegram. Book a test Calendly appointment → check Interview Notes was auto-created. Visit the site → check Visitor Log. Trigger all paths once and confirm everything lands correctly.
After Step 08 — system is live. Everything runs automatically from this point forward. You don't touch the infrastructure again unless something breaks or you want to add something new.
